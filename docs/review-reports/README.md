# Review Reports

This directory is for code audit, architecture review, and commit and PR readiness reports from `reviewer-agent`.

Use one Markdown file per task or issue when a durable report is requested.

Recommended filename:

- `<date>-<task-or-issue>-review-report.md`

Recommended sections:

- Review Result: APPROVE / REQUEST CHANGES
- Blocking issues
- Non-blocking suggestions
- Architecture notes
- Test coverage
- Commit / PR readiness

Reviewer output should lead with findings, ordered by severity and grounded in file and line references when available.
