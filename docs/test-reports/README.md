# Test Reports

This directory is for validation reports and manual QA evidence from `tester-agent`.

Use one Markdown file per task or issue when a durable report is useful.

Recommended filename:

- `<date>-<task-or-issue>-test-report.md`

Recommended sections:

- Test Result: PASS / FAIL
- Scope
- Test cases added
- Commands run
- Evidence
- Blocking issues
- Non-blocking issues
- Recommended fixes

The tester may add or update tests, but must not fix production source code.
